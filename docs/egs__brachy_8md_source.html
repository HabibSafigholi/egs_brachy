5 \section gen_info General Information
6 
7 egs_brachy is an egs++ application for rapid brachytherapy calculations for both photon and electron sources. The current documentation serves as a Technical Reference Manual to complement the egs_brachy user manual (https://clrp-code.github.io/egs_brachy/docs/egs_brachy_user_manual.pdf) and our initial egs_brachy paper (https://clrp-code.github.io/egs_brachy/docs/egs_brachy_paper2016.pdf): MJP Chamberland, REP Taylor, DWO Rogers, and RM Thomson, egs_brachy: a versatile and fast Monte Carlo code for brachytherapy, Phys. Med. Biol. 61, 8214-8231 (2016). Please cite this paper when egs_brachy is used in publications.
8 
9 egs_brachy was written by [Randle Taylor](http://randlet.com) in collaboration with Marc Chamberland, Dave Rogers, and Rowan Thomson of the [Carleton Laboratory for Radiotherapy Physics](http://www.physics.carleton.ca/clrp/).

51 \subsection runcontrol Run Control
52 
53 egs\_brachy uses the same run control block as other egs++ user codes
54 with the addition of one extra input `egsdat file format` which you
55 can use to tell egs\_brachy to output its egsdat file in gzip format.
56 Using gzip format can result in significantly smaller egsdat file sizes
57 for simulations with a large number of regions defined.
58 
59 \verbatim
60 :start run control:
61  ncase = 10000
62  nbatch=1
63  geometry error limit = 10
64  egsdat file format = gzip # gzip or text
65 :stop run control:
66 \endverbatim

69 \subsection runmodes Run Modes
70 
71 There are currently three different run modes available in egs_brachy:
72 
73 1. 'normal' (default): a regular simulation. For an example see [tests/seeds_in_xyz/seeds_in_xyz.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/seeds_in_xyz/seeds_in_xyz.egsinp).
74 2. 'superposition' : This type of simulation requires the used of an
75 EGS_ASwitchedEnvelope as the simulation geometry. At the start of every
76 history only a single inscribed geometry will be activated. This allows
77 you to explore the effects of intersource attenuation by performing TG-43
78 dose superposition type calculations. For an example see [tests/tg43mode/tg43mode.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/tg43mode/tg43mode.egsinp).
79 3. 'volume correction only': Just run the volume correction routines, print
80 the results and then quit. No actual dose calculations will be done. For an example see [tests/volume_correction/vc.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/volume_correction/vc.egsinp).
81 
82 
83 The run mode is set using a 'run mode' input block:
84 
85 \verbatim
86 :start run mode:
87 
88  run mode = normal # 'normal', 'superposition', or 'volume correction only'
89 
90 :stop run mode:
91 \endverbatim
92 
93 In simulations with more than a single source, egs_brachy uses the first source
94 as a particle generator. That is, all particles are initiated at the location
95 of the first source and transported until they escape the source geometry. From
96 there, the particle is moved to the location of the next source before
97 transport continues (more accurately, a new particle is added to the top of the
98 stack at the location of the chosen source and the original particle is
99 killed).
100 
101 This behaviour can be disabled by setting `single generator = no` in the run
102 mode block (see [tests/single_generator/single_generator.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/single_generator/single_generator.egsinp)).
103 
104 \verbatim
105 
106 :start run mode:
107  run mode = normal
108  single generator = no
109 :stop run mode:
110 
111 \endverbatim
112 
113 Disabling the single generator (see [tests/single_generator/multiple_generator.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/single_generator/multiple_generator.egsinp)) mode may be slightly faster for some simulations
114 (there is an extra call to howfar for every particle escaping the source
115  geometry in single generator mode). Note that in some situations (particle
116  recycling for example) single generator will always be enabled by egs_brachy.

118 \subsection geomspec Geometry Specification
119 
120 In general, arbitrary egs++ geometries can be included in a simulation
121 although, for the sake of efficiency, it is expected that EGS_AEnvelope geometries
122 will be used for most simulations.
123 
124 There are three egs_brachy specific input keys that are required for the
125 'geometry definition' input block (in addition to the standard egs++
126 'simulation geometry' key):
127 
128 1. 'source geometries' : this key specifies which geometries define the actual
129  brachytherapy source object. This may be a single geometry name (e.g. when
130  using the `egs_glib` shim to load an externally defined geometry) or a
131  list of all the sub-geometries used to compose a single source geometry when
132  definining geometries inline.
133 
134 2. 'phantom geometries': this tells egs_brachy which geometries to score dose
135  in (1 or more phantom geometries are required). Currently 3 geometry types
136  are allowed to be used as phantoms:
137  1. EGS_XYZGeometry (library egs_ndgeometry, see [tests/seeds_in_xyz/seeds_in_xyz.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/seeds_in_xyz/seeds_in_xyz.egsinp))
138  2. EGS_RZGeometry (library egs_rz, see [tests/stepped_source/stepped.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy../tests/stepped_source/stepped.egsinp))
139  3. EGS_cSpheres (library egs_spheres, see [tests/scatter/scatter.egsinp](../tests/scatter/scatter.egsinp))
140 
141 3. 'source envelope geometry': this input is only required for
142  superposition run mode and must name the EGS_ASwitchedEnvelope
143  geometry that contains the sources.
144 
145 Other phantom geometry types could be added in future provided they implement the
146 getBound, getNRegDir, and getMass methods of the EGS_BaseGeometry class.
147 
148 An (abbreviated) example geometry specification might look like:
149 \verbatim
150 :start geometry definition:
151 
152  # all geometries that are a part of the source
153  source geometries = planes, end_cap_1, seed_middle, end_cap_2, seed
154 
155  phantom geometries = phantom
156 
157  simulation geometry = phantom_w_seeds
158 
159  # only required for superposition run mode
160  source envelope geometry = phantom_w_seeds
161 
162  :start geometry:
163 
164  name = phantom
165  library = egs_ndgeometry
166  type = EGS_XYZGeometry
167 
168  # rest of geom definition
169 
170  :stop geometry:
171 
172  :start geometry:
173  library = egs_planes
174  name = planes
175  # ...
176  :stop geometry:
177 
178  :start geometry:
179  name = end_cap_1
180  #...
181  :stop geometry:
182 
183  :start geometry:
184  name = seed_middle
185  #...
186  :stop geometry:
187 
188  :start geometry:
189  name = end_cap_2
190  #...
191  :stop geometry:
192 
193  :start geometry:
194 
195  library = egs_cdgeometry
196  name = seed
197  base geometry = planes
198  set geometry = 0 end_cap_1
199  set geometry = 1 seed_middle
200  set geometry = 2 end_cap_2
201 
202  :stop geometry:
203 
204  :start geometry:
205 
206  library = egs_autoenvelope
207  name = phantom_w_seeds
208  base geometry = phantom
209 
210  :start inscribed geometry:
211 
212  inscribed geometry name = seed
213 
214  # rest of auto envelope inscribed geom definition
215 
216  :stop inscribed geometry:
217 
218  :stop geometry:
219 
220 :stop geometry definition:
221 
222 \endverbatim

224 \subsubsection ctdata Using CT data to create phantoms
225 
226 Using the `egs_glib` geometry library you can construct an `EGS_XYZGeometry`
227 using a `.egsphant` file like so:
228 
229 \verbatim
230 
231 :start geometry:
232  library = egs_glib
233  type = egsphant
234  name = my_egsphant_geom
235  egsphant file = /path/to/some/egsphant/file
236  density file = /path/to/density/file
237 :stop geometry:
238 
239 \endverbatim
240 
241 The egsphant file may either be a typical .egsphant text file or gzipped .egsphant.gz file, and the latter compressed format is often advantageous in terms of memory) The density file indicates to egs_brachy the nominal density of each medium in the egsphant (voxel-by-voxel densities are defined in the egsphant file), and currently egs_brachy can read these data from a (a) material.dat file, (b) pegs4 data file, or (c) a simple text file of the format:
242 
243 \verbatim
244 
245 MEDIUM=WATER
246 RHO=1.000
247 MEDIUM=AIR
248 RHO=1.2E-3
249 
250 \endverbatim
251 
252 
253 *Note the egs_glib geometry library in the main EGSnrc repository does not currently
254 contain support for egsphant files. The egs_glib contained in the egs_brachy git branch is
255 customized to include support for egsphant files.*
256 
257 It is also possible to use the egs_ndgeometry library to contstruct an `EGS_XYZGeometry`
258 using a `.egsphant` file like so:
259 
260 \verbatim
261 
262 :start geometry:
263 
264  library = egs_ndgeometry
265  type = EGS_XYZGeometry
266  name = my_egsphant_geom
267  egsphant file = /path/to/some/egsphant/file
268  ct ramp = /path/to/ramp/file.ramp
269 
270 :stop geometry:
271 
272 \endverbatim
273 
274 See the `egs_ndgeometry` documentation for details about the format
275 of the ct ramp file. Note, however, that the egs_ndgeometry approach assumes that there is no overlap in voxel mass densities for different media in the egsphant, i.e., each medium has a distinct and non-overlapping range of mass densities. This is often *not* the case for brachytherapy phantoms, and so using the `egs_glib` geometry with `.egsphant` files (gzip format) is recommended.

286 \subsection scoringopts Scoring options
287 
288 The 'scoring options' input block currently has the following keys:
289 
290 - 'muen file': path to the file containing the muen data required for the
291  simulation. See the MuenDataParser class for more information about the
292  format of this file.
293 - 'muen for media': (Optional) a list of the materials that muen data is required for.
294  Dose will be set to 0 for any phantom media that don't have muen data
295  available. (Can be used instead of or in conjunction with `muen for medium`).
296 
297 - 'muen for medium': (Optional) one or more input blocks which specify the muen dataset
298  to be used for a given transport medium. (Can be used instead of or in conjunction
299  with `muen for media`) This allows you to, for example, transport particles in tissue
300  but score in water with notation Dw,m according to the conventions of TG-186.
301 
302 \verbatim
303  :start muen for medium:
304  transport medium = WATER
305  scoring medium = TISSUE
306  :stop muen for medium:
307 \endverbatim
308 
309 - 'score tracklength dose' : (Optional) Controls whether dose using a tracklength
310  estimator is scored in the phantom geometries. The choices are 'yes' (default) or
311  'no'.
312 
313 - 'score energy deposition' : (Optional) Controls whether dose from particle
314  interactions is scored in the phantom geometries. The choices are 'yes' or
315  'no' (default). See
316  [tests/brem_cyl/brem_cyl.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/brem_cyl/brem_cyl.egsinp) for an
317  example
318 
319 - 'score scatter dose' : (Optional) Controls whether scatter dose (normalized
320  to total radiant energy) is scored in the phantom geometries. The choices are
321  'yes' or 'no' (default). See
322  [tests/scatter/scatter.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/scatter/scatter.egsinp) for an example.
323 
324 - 'spectrum scoring': (Optional) zero or more input blocks specifying spectra
325  to score (described below). See
326  [tests/spec_absolute/spec_absolute.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/spec_absolute/spec_absolute.egsinp),
327  [tests/spec_eflu/spec_eflu.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/spec_eflu/spec_eflu.egsinp), and
328  [tests/spec_vox/spec_vox.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/spec_vox/spec_vox.egsinp) for
329  examples.
330 
331 - 'phsp scoring': (Optional) zero or one input block specifying whether to
332  score a phase space on the surface of the source (described below). See
333  [tests/phsp_scoring/phsp_score.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/phsp_scoring/phsp_score.egsinp)
334  for an example.
335 
336 - 'dose file format': (Optional) Controls whether 3dddose files are
337  written as text or gzipped text files. Options are 'text' (default) and 'gzip'.
338 
339 - 'output egsphant files': (Optional) Controls whether egsphant files are
340  created for scoring phantoms. Options are 'no' (default) and 'yes'.
341 
342 - 'egsphant file format': (Optional) Controls whether egsphant files are
343  written as text or gzipped text files. Options are 'text' (default) and 'gzip'.
344 
345 - 'output voxel info files': (Optional) Controls whether voxel info files are
346  created for scoring phantoms. Options are 'no' (default) and 'yes'. See below
347  for a description of these files. See
348  [tests/volume_correction/vc.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/volume_correction/vc.egsinp) for
349  an example.
350 
351 - 'voxel info file format': (Optional) Controls whether voxel info
352  files are written in text (default) or gzip format
353 
354 - 'output volume correction files for phantoms': (Optional) Controls which phantoms
355  will have volume correction files created.
356 
357 - 'volume correction file format': (Optional) Controls whether volume correction
358  files are written in text (default) or gzip format
359 
360 - 'record initial particle positions': (Optional) If this is set to a number N,
361  where N > 0, then the first N initial particle positions will be written to
362  a .pinit file. You can use this file to visualize the initial particle positions
363  using gnuplot for example.
364 
365 - 'current result phantom region': (Optional) Controls which phantom region is
366  used for displaying results at the end of each batch and/or terminating the
367  calculation if the required statistical uncertainty is reached. Defaults to
368  region 0 of the first phantom object. Currently the result of the
369  tracklength dose scoring is always used. See
370  [tests/brem_cyl/brem_cyl.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/brem_cyl/brem_cyl.egsinp) for an
371  example.
372 
373 - 'dose scaling factor': (Optional) Allows you to scale all dose files output by
374  egs_brachy by a constant factor.
375

422 \subsubsection vcfiles Volume correction files
423 
424 If one or more phantoms are specfied in 'output volume correction files for
425 phantoms' then egs\_brachy will output a file containing region numbers and the
426 corrected volumes for any regions which has had its volume corrected.
427 
428 File mode is specified with the 'volume correction file format' input and can either
429 be 'text' or 'gzip'.
430 
431 These volume correction files allow you to precompute volume corrections for a
432 given geometry arrangement and can then be used in conjunction with the 'volume
433 correction from file' volume correction option. This can be particularly
434 useful when running a simulation in parallel. A single job in "volume
435 correction only" mode can be run to initially calculate the volume corrections
436 and then the parallel jobs can use the precomputed volume corrections values
437 eliminating the redundant calculation of volume corrections by every job in
438 your parallel run.
439 
440 The input file
441 [tests/volume_correction/vc.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/volume_correction/vc.egsinp)
442  demonstrates this feature.

446

493 \subsubsection phspscoring Phase Space Scoring Options
494 
495 egs\_brachy has the ability to score an IAEA format phase space file of all
496 particles escaping the source geometry. This phase space can then be used as a
497 particle source in future simulations with the eb_iaeaphsp_source source type
498 (see Phase Space Source section below).
499 
500 To enable this option you need to add a `phsp scoring` block to the `scoring
501 options` input section. The `phsp scoring` block has the following inputs:
502 
503 1. 'phsp output directory': (Optional) Controls where the .IAEAheader & .IAEAphsp files
504  are written too. Defaults to the current working directory.
505 
506 2. 'access mode': (Optional) Controls whether egs_brachy should write to a new file or
507  append to an existing phsp file. Options are `write` and `append` (default).
508 
509 3. 'kill after scoring': (Optional) Controls whether the weight of scored
510  particles should be set to 0 which will increase the speed of phase space
511  file generation. Options are `yes` and `no` (default).
512 
513 4. 'boundary step': (Optional) Since egs\_brachy scores phsp particles on the
514  surface of a source, you may find that when you use a phsp source,
515  particles get stuck at the geometry boundary when they are initialized. To
516  combat this, before scoring a phsp particle egs\_brachy propagates the particle
517  a small distance forward along its current direction of travel before writing it
518  to the phase space file. By default this value is 1E-4 cm but you can make
519  it smaller or larger if necessary.
520 
521 5. 'print header': (Optional) Controls whether the iaea_print_header function
522  is called during the outputResults. Options are `yes` and `no` (default).
523 
524 A complete phsp scoring block looks like:
525 
526 \verbatim
527 
528  :start phsp scoring:
529  phsp output directory = /home/randlet/egs/egsnrc/egs_brachy/
530  access mode = write
531  print header = no
532  kill after scoring = yes
533  boundary step = 1E-6
534  :stop phsp scoring:
535 
536 \endverbatim
537 
538 See
539 [tests/phsp_scoring/phsp_score.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/phsp_scoring/phsp_score.egsinp)
540 for an example of scoring a phase space and
541 [tests/phsp_run/phsp_run.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/phsp_run/phsp_run.egsinp) for an
542 example of running a phase space source.

549 \subsection sourcedef Source definition
550 
551 The `source definition` block for egs_brachy consists of 3 parts:
552 
553 1. a standard egs++ `source` block which defines what the base particle source
554  and spectrum will be (typically for brachy calculations this will be an
555  isotropic source).
556 2. a `transformations` input block which contains one or more
557  EGS_AffineTransform input blocks to tell egs_brachy the location of all
558  particle sources (this will usually be identical to the `inscribed geometry
559  -> transformations` block for an egs_autoenvelope geometry and hence an
560  external file and an `include file` directive should probably be used.)
561 3. a `simulation source` input item which is set to the name of the source
562  defined in 1.
563 4. an optional `source weights` input item which is a list of relative
564  statistical weights for each source. This input is described below in the
565  `Variable source weights` section.
566 
567 
568 An example `source definition` block is shown below for a 6702 seed:
569 \verbatim
570 
571 :start source definition:
572 
573  :start source:
574 
575  library = egs_isotropic_source
576  name = 6702_spheres
577  charge = 0
578 
579  # three spherical shell sources for 6702 Source
580  :start shape:
581 
582  library = egs_shape_collection
583 
584  :start shape:
585  library = egs_spherical_shell
586  inner radius = 0.0299
587  outer radius = 0.03
588  midpoint = 0, 0, -0.11
589  :stop shape:
590 
591  :start shape:
592  library = egs_spherical_shell
593  inner radius = 0.0299
594  outer radius = 0.03
595  midpoint = 0, 0, 0
596  :stop shape:
597 
598  :start shape:
599  library = egs_spherical_shell
600  inner radius = 0.0299
601  outer radius = 0.03
602  midpoint = 0, 0, 0.11
603  :stop shape:
604 
605  probabilities = 1 1 1
606 
607  :stop shape:
608 
609  :start spectrum:
610  type = tabulated spectrum
611  spectrum file = /home/randlet/egs/HEN_HOUSE/spectra/I125_TG43.spectrum
612  :stop spectrum:
613 
614  :stop source:
615 
616  :start transformations :
617  :start transformation:
618  translation = -2,-2,-2
619  :stop transformation:
620  :start transformation:
621  translation = -2,-2,-1
622  :stop transformation:
623

634 \subsubsection varweight Variable source weighting
635 
636 In order to simulate sources with different activity levels you can add a
637 `source weights` input to the `source definition` section. For example, if you
638 have a simualtion with two sources, the first of which has three times the
639 activity of the second, you would set the following source input:
640 
641 \verbatim
642 
643 :start source definition:
644 
645  :start source:
646 
647  library = egs_point_source
648  name = pt_source
649  charge = 0
650  position = 0 0 0
651 
652  :start spectrum:
653  type = tabulated spectrum
654  spectrum file = /home/randlet/egs/HEN_HOUSE/spectra/I125_TG43.spectrum
655  :stop spectrum:
656 
657  :stop source:
658 
659  :start transformations :
660  :start transformation:
661  translation = 0 0 -1
662  :stop transformation:
663  :start transformation:
664  translation = 0 0 1
665  :stop transformation:
666  :stop transformations:
667 
668  simulation source = pt_source
669 
670  source weights = 3 1 # give source at (0, 0, -1) three times the weight of source 2
671 
672 :stop source definition:
673 
674 \endverbatim
675 
676 egs_brachy uses these weights to assign the initial statistical weight of the
677 particles originating in a source. If the `source weights` input is missing
678 all sources are given equal weighting. If the number of inputs for source
679 weights is less than the number of sources, the missing inputs will be assumed
680 to be 1. See
681 [tests/variable_activity/variable.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/variable_activity/variable.egsinp)
682 for an example.
683 
684 When the `superposition` run mode is used, the `source weights` input
685 represents relative dwell times (e.g. for a stepped Ir192 HDR source) rather
686 than different activity levels. See
687 [tests/stepped_source/stepped.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/stepped_source/stepped.egsinp)
688  for an example of this feature.

764 \subsubsection recycling Particle Recycling
765 
766 egs_brachy has the ability to reuse particles which escape from the source
767 geometry an arbitrary number of times. With particle recycling enabled,
768 egs_brachy detects (in ausgab) when a particle is escaping the source, and then
769 adds Nrecycle new particles to the top of the stack at the location of
770 each source in the simulation.
771 
772 This can increase the efficiency of a
773 simulation where only a fraction of particles are escaping the source geometry
774 (since particles which don't escape the source geometry are 'wasted' because
775  they don't contribute to dose in the phantom).
776 
777 In order to enable source particle recycling you must include a `particle recycling`
778 block in the `variance reduction` block.
779 
780 \verbatim
781 
782 
783 :start variance reduction:
784  :start particle recycling:
785  times to reuse recycled particles = 10
786  rotate recycled particles = yes
787  :stop particle recycling:
788 :stop variance reduction:
789 
790 \endverbatim
791 
792 If `rotate recycled particles` is set to yes, each particle will be rotated
793 by an arbitrary angle about the z-axis prior to being reused. (If
794 `times to reuse recycled particles` is greater than 1, particle rotation is
795 enforced).
796 
797 Examples of particle recycling may be seen in the following test files:
798 [tests/phsp_run/phsp_run.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/phsp_run/phsp_run.egsinp),
799 [tests/recycling/recycling.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/recycling/recycling.egsinp),
800 [tests/tg43mode_recycle/tg43mode_recycling.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/tg43mode_recycle/tg43mode_recycling.egsinp),
801 [tests/variable_w_recycling/variable_w_recycling.egsinp](https://github.com/clrp-code/egs_brachy/blob/master/egs_brachy/tests/variable_w_recycling/variable_w_recycling.egsinp).
802 
803 
804 #### Particle Recycling with PHSP Sources
805 
806 Phase space particles are scored once they've already escaped the source and
807 therefore particles from a phsp source will be initiated outside of sources.
808 Because of this phsp particles never escape the source and trigger the
809 recycling routines in ausgab. Instead, at the beginning of each history a
810 single particle is retrieved from the phase space file and then NRecycle copies
811 of the particle are made and placed at the location of all the sources (so the
812  history starts with NRecyle\*NSource particles on the stack rather than
813  just a single particle).
814 
815 #### Particle Recycling and Superposition Mode
816 
817 Currently when running in superposition mode recycled particles are only
818 generated for the currently active source rather than all source locations (
819  i.e. when a particle escapes a source only `Nrecycle` particles are created
820  rather than `Nrecycle*Nsource`).
821 
822 \subsubsection rangerejection Range Rejection
823 
824 Range rejection is enabled outside of sources and disabled within
825 sources by default in egs_brachy. The default maximum range rejection
826 energy is set to 2.511 MeV (including rest mass). In other words, by
827 default range rejection is applied to electrons in the phantom with
828 kinetic energy lower than 2 MeV. To enable or disable range rejection
829 within sources, or outside of sources, use the `source range rejection`
830 and `global range rejection` inputs, respectively.
831 
832 The `source range rejection max energy` and `global range rejection max
833 energy` inputs are used to set the maximum energy of electrons (in MeV,
834  including rest mass) to use range rejection with.
835 
836 \verbatim
837 
838 :start variance reduction:
839  global range rejection = yes
840  global range rejection max energy = 0.611
841  source range rejection = yes
842  source range rejection max energy = 0.516
843 :stop variance reduction:
844 
845 \endverbatim

879 \subsection voxvc Voxel volume correction details
